# Prototype Implementation and Front-End Development

## Problem Context

After defining the functional requirements, structured feedback model, interaction flow, and content structure, the next stage was translating the design into a working prototype.

The aim was not only to create static screens, but to build an interactive prototype that allowed users to move through the main community feedback journey.

This meant turning the wireframe into connected pages with clear navigation, consistent styling, and basic interaction feedback.

## Design-to-Code Process

The implementation began with the wireframes created during the design stage.

These wireframes helped define:

- Page layout
- Content hierarchy
- Navigation flow
- Feedback input areas
- Image placement

The prototype was then developed using the provided Bla + Bla Corp project template. The implementation used HTML templates for structure, CSS for visual styling, and Mojo routing to connect pages.

The final prototype focused on creating a working version of the main interaction flow rather than implementing a full backend system.

## Implemented Pages

### Home Feed

The home feed presents multiple community posts in a card-based layout.

Each post includes:

- Project image
- Post title
- Feedback goal
- Category
- Date
- View and feedback link

This page supports browsing and discovery, allowing users to quickly understand what kind of feedback each post is requesting.

### Detail Pages

Each post links to a separate detail page.

The detail pages include:

- A large project image
- Creator information
- Project description
- Structured feedback prompts
- Existing comments

This supports the reviewing journey by giving users enough context before they provide feedback.

### Create Post Page

The create post page allows users to frame their own feedback request.

It includes fields for:

- Post title
- Feedback goal
- Project description
- Prototype or image upload

This page supports the content creation side of the community hub.

### Feedback Goal Checklist

The checklist page was added as a second step after the create post page.

Its purpose is to help users reflect on whether their feedback request is clear before publishing.

This supports the project goal of improving feedback quality before the post enters the community feed.

## Interaction Flow Implemented

The final prototype supports two main flows:

**Home Feed → Detail Page → Structured Feedback**

**Home Feed → Create Post → Feedback Goal Checklist → Publish Post**

These flows match the interaction model developed in earlier posts and demonstrate how the platform supports both reviewers and content creators.

## Technical Challenges

Several technical issues appeared during implementation.

One challenge was connecting pages correctly through routes and controller actions. Some pages initially failed to load because the route name, controller action, and template file name did not match.

This was resolved by keeping naming consistent across the project.

Another challenge was styling consistency. Early versions of the interface looked too close to default browser styling, so the CSS was revised to create a more polished visual system using cards, spacing, rounded corners, and a consistent purple-blue accent colour.

Image display was also adjusted so that prototype images appeared correctly inside post cards and detail pages.

## Design Outcomes

The final prototype successfully translated the wireframe into a working browser-based experience.

Users can now:

- Browse community posts
- Open different post detail pages
- View prototype images
- Enter structured feedback
- Create a new post
- Complete a feedback goal checklist before publishing

This demonstrates the main concept of a goal-oriented feedback community.

## Limitations

The prototype does not yet store new posts or feedback dynamically.

Some interactions are simulated rather than fully connected to a database. This is appropriate for the current prototype scope, as the focus is on demonstrating the interaction model and user journey.

Future development could include dynamic data storage, real user profiles, and live comment submission.

## Conclusion

This implementation stage transformed the project from a design concept into an interactive prototype.

By combining HTML templates, CSS styling, and routing, the prototype demonstrates how structured, goal-oriented feedback can be supported through a clear multi-page user experience.
