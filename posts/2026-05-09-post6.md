# Evaluation, Reflection, and Future Development

## Project Goal

The goal of this project was to design a community platform that improves the quality of peer feedback.

Instead of allowing feedback to remain broad or disconnected from the poster’s intention, the prototype encourages users to define a clear feedback goal and respond through structured prompts.

The final Assignment 2 prototype supports this goal through a connected multi-page interaction flow.

## Prototype Evaluation

The final prototype supports two main user journeys.

### Journey 1: Reviewing Community Posts

Users can browse the home feed, open a post detail page, view the prototype image, read the project context, and provide structured feedback.

The structured feedback prompts guide users to focus on:

- What feels clear
- What feels confusing
- What could be improved

This makes feedback more focused and easier for the original poster to interpret.

### Journey 2: Creating a New Post

Users can create a new post by entering a title, feedback goal, project description, and prototype image.

They then move to the Feedback Goal Checklist before publishing.

This extra step encourages users to reflect on whether their request is clear and specific before sharing it with the community.

## What Worked Well

The most successful design decision was connecting the feedback goal with the feedback form.

The home feed introduces each post through its title, image, category, and goal.

The detail page then provides enough context before asking users to respond.

This creates a clearer relationship between what the poster needs and what the reviewer provides.

The multi-page structure also worked well. Separating the home feed, detail page, create post page, and checklist page reduced visual overload and made each task easier to understand.

## Limitations

The prototype still has several limitations.

First, feedback submission is simulated rather than stored dynamically. Users can experience the interaction flow, but the system does not yet save new feedback.

Second, the create post flow does not yet add a new post to the home feed automatically.

Third, the platform does not yet support filtering, sorting, or saved posts, which may affect scalability in larger communities.

## Accessibility Considerations

Accessibility was considered through:

- Clear page structure
- Labelled form fields
- Large clickable buttons
- Consistent spacing and visual hierarchy

Future improvements should include:

- Better keyboard navigation
- More detailed image alt text
- Clear focus states
- Additional colour contrast testing

## Future Development

Future development could include:

- Persistent data storage
- Live feedback updates
- Search and filtering
- User profiles
- Feedback history

These features would improve both usability and long-term community engagement.

## Reflection

This project helped me understand that designing a community platform is not only about interface design, but also about designing relationships between users, content, and interaction.

The most important design decision was requiring users to define a feedback goal before receiving responses.

This decision shaped the entire prototype, from the home feed to the detail page and create post flow.

## Conclusion

The final prototype successfully demonstrates the concept of a goal-oriented feedback community.

By combining goal-based posting, structured feedback prompts, and a checklist before publishing, the platform supports more focused, contextual, and actionable community interaction.
