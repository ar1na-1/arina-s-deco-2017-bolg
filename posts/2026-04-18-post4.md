# Designing the System Structure and Content Model

## Problem Context

After defining the interaction flow of the platform, the next challenge was determining how content should be organised across different pages.

In a goal-oriented community system, users are not only browsing content, but also interpreting project goals, reviewing prototypes, and providing structured feedback.

If the system structure is unclear or inconsistent, users may struggle to understand what each post represents or how different pieces of feedback relate to the original goal.

This means the platform requires a content model that supports both clarity and consistency across the entire interaction flow.

## Design Objective

The objective of this stage was to define what information each page needs in order to support meaningful interaction.

Rather than designing a complex backend system, the prototype focuses on a lightweight content structure that supports the core user journey:
