# Defining Functional Requirements for a Goal-Oriented Feedback Community

## Problem Context

Online communities allow users to share ideas, projects, and work-in-progress content with others. However, the feedback users receive is often broad, subjective, or disconnected from what they actually want to improve.

This becomes a problem when users are not simply looking for general opinions, but need feedback on a specific goal, such as improving navigation clarity, testing visual hierarchy, or understanding whether an interaction feels intuitive.

## Problem Analysis

The main issue is not the lack of feedback, but the lack of context. When reviewers do not understand the poster’s intended focus, their responses may become misaligned with the original purpose of the post.

For example, a user may want feedback on whether a task flow is easy to follow, but receive comments about colours or visual style instead. Although these comments may be valid, they do not directly help the user evaluate the issue they are trying to solve.

This reduces the usefulness of feedback and makes it harder for users to identify clear next steps.

## Proposed Direction

This project proposes a lightweight community hub where users can share work, ideas, or early-stage projects and define a clear feedback goal when posting.

Instead of treating all posts as open-ended discussions, the system encourages users to frame what kind of feedback they need. Reviewers can then respond with more focused and relevant comments.

The goal is not to build a full social media platform, but to design a community interaction model that improves the quality and usefulness of peer feedback.

## Core Functional Requirements

The system must allow users to create a post with a title, description, and clear feedback goal. The feedback goal is essential because it gives reviewers context before they respond.

The system must display each post in a feed with its associated goal. This allows users to quickly understand what kind of feedback is being requested before opening the post.

The system must provide a detailed post page where reviewers can view the full context before giving feedback. This prevents users from responding too quickly without understanding the post.

The system must guide reviewers through structured feedback prompts, such as what feels clear, what feels confusing, and what could be improved. These prompts help make feedback more specific and actionable.

The system must support a create-post flow that helps users clarify their feedback goal before publishing. This ensures that the request is focused before it enters the community feed.

## Optional Features

Additional features could include exploring related ideas, filtering posts by topic, or viewing feedback patterns over time. These features may improve discoverability and reflection, but they are not essential to the core concept.

For this prototype, the priority is to demonstrate the main interaction flow: browsing posts, understanding feedback goals, providing structured feedback, and creating a new goal-oriented post.

## Scope Decisions

Complex features such as expert matching, reputation systems, automated analysis, and advanced user profiles are excluded from the initial scope. While these could add value in a larger platform, they would increase implementation complexity and distract from the core interaction model.

The prototype therefore focuses on a small set of pages and interactions that directly support goal-oriented feedback.

## Conclusion

The core requirement of this project is to make feedback more contextual, focused, and actionable within a community setting. By requiring posters to define a clear goal and guiding reviewers through structured prompts, the system aims to improve the quality of peer feedback without becoming overly complex.
